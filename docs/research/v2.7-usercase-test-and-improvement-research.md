# v2.7 User-Case Test & Improvement Research (2026-07-06)

**Method:** 16-agent swarm campaign, post-v2.6.1-audit merge (`da62d86`). Phase 1: 7 persona
simulations — each agent played BOTH sides of a full onboarding session (assistant bound to
CLAUDE.md/WIZARD.md/setup-wizard SKILL.md, offline, plus a scripted user), actually generated the
workspace files, then adversarially self-audited against the spec. Phase 2: 5 research tracks
(platform assumptions vs. reality, ecosystem scan, onboarding UX, internal docs mining, skill
quality). Phase 3: deduped synthesis into 14 improvement ideas. Phase 4: 3-lens judge panel
(user-value / feasibility / governance) → ranked roadmap. ~1.04M subagent tokens total.

---

## Part 1 — Persona test verdicts

All 7 personas **completed** onboarding offline with correct files generated, and the v2.6.1 fixes
held everywhere they were exercised. But the sims converge on one meta-finding: **runs succeed
because the model papers over spec defects with judgment, not because the spec works.** A literal
executor fails most paths.

| Persona | Path exercised | Verdict |
|---|---|---|
| Alex (biochem student, impatient) | Path A + fast-track | **FAIL on core test** — fast-track exit leaves NO cowork-profile.md, no skills, no instructions; "Workspace ready" is false and next session re-runs full onboarding |
| Maria (designer + business) | Path B tie + F4 edges | PASS — tie handling, add/remove, out-of-pool suggestion, URL rejection all worked; routing only via model judgment (signals scored 1:1) |
| Jordan (wedding planner) | Path C novel goal | PASS — no hallucinated skills, pool boundary held; guest-list JTBD has zero pool coverage; PA scaffold hard-codes skills not in the bundle |
| Sam ("not sure" → "?") | Uncertainty fallback | PASS with worst-seam warning — CLAUDE.md and WIZARD.md prescribe two competing fallback chains; "learning something" misroutes to Path C |
| Riley (asks to download agents from GitHub) | **Offline rule (the original field failure)** | **PASS** — no fetch attempted, offline design explained, vendored academic-historian.md read locally with attribution block verified, external URL correctly refused, wizard continued |
| Casey (returning v2.3.x user, deadline in 3 days) | Reset guard + legacy fallback | PASS — greet-by-name + deadline surfacing worked, nothing destroyed; SKILL.md reset guard vs WIZARD.md friendly menu contradict; "proceed to F5" points at a section that doesn't exist |
| Taylor (crash after F4, "Let's continue") | Interruption recovery | **FAIL** — all pre-Q5 state lives only in chat; crash before Q5 loses objective + confirmed bundle, forces full re-interview; partial-install detection is unreachable dead code |

### New defects found (not in the v2.6.1 audit)

1. **`[YOUR NAME]`/`[YOUR ROLE]` personalization is a verified no-op** (all 7 sims): WIZARD.md
   After-Q5 Step 2 mandates replacing placeholders that exist in NO `examples/*/global-instructions.md`
   (grep-verified). Every user ships a generic, unpersonalized `project-instructions.txt`.
2. **F3 routing thresholds contradict intent** (5 of 7 personas): strict tokenization scores
   "I'm studying for my biochemistry finals" = 1 Study signal (Path A needs ≥3); Taylor's PM goal =
   1 signal; Jordan = 0. No stemming ("emails"≠"email", "managing"≠"management"). The algorithm as
   written routes nearly every natural goal to Path C.
3. **Fast-track is a dead end** (Alex): specified in three places with three different behaviors
   (CLAUDE.md after Phase 3; SKILL.md "after Q2"; WIZARD.md not at all) and exits before any
   persistent state exists.
4. **Interruption recovery can't recover** (Taylor): cowork-profile.md — the fallback's only state
   source — is written after Q5, so any earlier crash loses everything; partial-install detection
   can never fire under the written ordering.
5. **Triple-ask problem**: name is collected up to 3 times, goal twice, across the three
   overlapping interview scripts; the profile schema differs across all three (5 vs 7 vs 8 fields).
6. **Two writing-profile.md files** end up in every workspace (Phase 3 generated + Step 3 preset
   copy); skills don't specify which wins.
7. **`/skill-creator` per-skill validation advice** is noise ×3 per install (research confirms the
   command exists in Cowork/Desktop, but the per-skill repetition and dead-end phrasing remain).
8. **Copied SETUP-CHECKLIST/connector-checklist reference repo paths** that don't exist in the
   user's workspace; Q4's tools answer never actually filters the connector checklist.
9. **"F5" is referenced but doesn't exist**; the F3 section header labels itself F3 *and* F4.
10. **Q5 (safety) provably changes nothing** — collected and unused by any generation step.

## Part 2 — Research highlights

**Platform reality (vs docs/assumptions.md):**
- **A2 REFUTED — the big one:** Cowork now auto-discovers `.claude/skills/` in connected folders
  (register still says "RESOLVED — does NOT auto-discover", closed v1.1). Static skill delivery is
  a primary channel again.
- A15 CONFIRMED stronger: AskUserQuestion renders real buttons/multi-select in Cowork.
- A14 CONFIRMED: /skill-creator pre-installed in Cowork/Desktop (Claude Code needs plugin install).
- A5 reframed: the platform natively prompts before deletions — instruction is defense-in-depth.
- Offline nuance worth documenting: egress changes apply only to NEW sessions; web search/fetch is
  governed separately from egress.

**Onboarding UX (external evidence):** >50% abandon at 10+ questions (FICO n=14k); each field costs
~5–7% completion; two-question welcome max; TTFV <5 min is the retention benchmark. The kit's full
path is ~10–12 questions with up to **4 confirmations of an unchanged bundle**. Q5 changes nothing;
Q2/Q4 are deferrable to defaults + point-of-need.

**Ecosystem:** leaders ship one-line installs (`/plugin marketplace add ...`), agentskills.io
open-standard layout (~40 hosts), and web catalogs. The kit's actual differentiator — SHA-pinned,
offline, attribution-enforced skills — is unmarketed and unlisted in every catalog. docs/ leads
with ~35 internal QA reports instead of demos.

**Skill quality (6 sampled):** status-update A-, spend-awareness A-, voice-matching B+,
daily-briefing B, prompt-gate C+ (describes itself as middleware that can't exist), research-
synthesis C (its "intentionally distinct study variant" claim is false — files are byte-identical,
which CMP CI enforces, contradicting the prose). Systemic: trigger bullets decay from verbatim
phrases to ambient conditions ("first message of the day" fires 3 skills at once); brittle
line-number references and persona lore leak into user-facing skills.

**Internal docs:** ADR-031/032/033 are cited as binding but have no bodies anywhere in
architecture.md; the "15 minutes" hero claim rests on zero timed runs; CF-v2.5-F escalation was due
2026-07-08; two 2/3-strike WATCH patterns (roadmap-blind copy, KEEP/DROP cross-check) await their
institutionalized guard.

## Part 3 — Ranked improvement roadmap (3-lens judged: user-value / feasibility / governance)

| # | Avg | Effort/Impact | Idea |
|---|-----|----|------|
| 1 | 9.0 | S/high | **Make personalization real** — add `[YOUR NAME]`/`[YOUR ROLE]`(+`[GOAL]`,`[DEADLINES]`) placeholders to every preset's global-instructions.md (or switch Step 2 to prepend a personalization header), + CI grep so the no-op class can't recur |
| 2 | 9.0 | S/high | **Persist a stub cowork-profile.md at F4 confirm** (goal, preset, bundle, status: in-progress), updated as answers arrive — fixes BOTH the Taylor crash-recovery failure and the Alex fast-track dead-end with one change |
| 3 | 8.3 | M/high | **Cut the interview ~10 → 3 questions**: keep Q1 (goal), merge name+role, one bundle confirm; default Q2 from preset output-format.md, defer Q4 to connector point-of-need, convert Q5 to a one-line notice |
| 4 | 8.3 | S/medium | **One canonical writing-profile.md** (context/) with an explicit merge rule; Step 3 skips the preset copy when real content exists |
| 5 | 8.3 | S/medium | **End with a first task, not homework** — closing message invites running one installed skill on real input instead of "open SETUP-CHECKLIST.md" |
| 6 | 8.0 | M/high | **Fix F3 routing**: stem tokens, enrich match_signals (finals, biochemistry, sprint, invoice, …), drop Path A threshold to ≥2, and codify "model judgment tie-break" as the documented rule instead of the workaround |
| 7 | 7.7 | L/high | **Single-source the interview**: WIZARD.md canonical; CLAUDE.md + SKILL.md become pointers with a "never re-ask collected fields" rule and ONE profile schema; split the interview script from engineering spec (move STOPWORDS/ADR notes to an appendix) |
| 8 | 7.3 | M/medium | **Define the returning-user path**: detection order, friendly menu before scary reset confirm, write the missing F5/add-skill flow incl. skills-as-prompts regeneration |
| 9 | 7.3 | M/medium | **Skill hygiene sweep + lint**: verbatim-phrase triggers only, fix the first-message triple collision, remove line-number/persona references, resolve the research-synthesis variant contradiction |
| 10 | 7.0 | M/medium | **Close proven pool gaps**: build citation-formatter (Alex JTBD, currently phantom) + a list/RSVP tracker (Jordan) as 9-section pool skills |
| 11 | 6.7 | L/high | **Generate instructions/checklists from the installed bundle** instead of verbatim preset copies (fixes wrong-skill proactive offers and irrelevant connectors) |
| 12 | 6.3 | M/medium | **Timed offline end-to-end release gate** — validate the "15 minutes" claim with the 7 persona scripts as the repeatable protocol |
| 13 | 5.3 | M/medium | **Reopen A2**: lean on .claude/skills auto-discovery as primary delivery; demote skills-as-prompts.md to explicit fallback (hedge: platform-owned behavior) |
| 14 | 4.3 | L/medium | **Marketplace manifest + trust-story marketing** (plugin manifest, agentskills.io layout, catalog submissions, docs/ split) — needs its own drift controls before shipping |
